Java 的继承与接口

2026 年 5 月 11 日

什么是继承？

继承（Inheritance）是面向对象编程的三大特性之一。 它允许一个类复用另一个类的属性和方法，从而避免重复代码，建立类之间的层次关系。

在 Java 中，被继承的类称为父类（Super Class / Parent Class）， 继承别人的类称为子类（Sub Class / Child Class）。 子类自动拥有父类所有的非私有成员，还可以添加自己独有的属性和方法。

打个比方：父类是「动物」——有名字、年龄，能吃东西、睡觉；子类是「狗」——在动物的基础上，还多了摇尾巴、汪汪叫。

extends 关键字

使用 extends 关键字让一个类继承另一个类：

// 父类
public class Animal {
    String name;
    int age;

    public void eat() {
        System.out.println(name + " 正在吃东西");
    }

    public void sleep() {
        System.out.println(name + " 正在睡觉");
    }
}

// 子类
public class Dog extends Animal {
    public void bark() {
        System.out.println(name + " 汪汪叫！");
    }
}

Dog 虽然没有定义 name、age、eat()、sleep()，但通过继承，它自动拥有了这一切：

Dog d = new Dog();
d.name = "旺财";
d.age = 3;
d.eat();   // 旺财 正在吃东西
d.sleep(); // 旺财 正在睡觉
d.bark();  // 旺财 汪汪叫！

Java 是单继承

Java 规定一个类只能有一个直接父类。下面的写法是错误的：

// 错误！Java 不支持多继承
public class Dog extends Animal, Pet { }  // 编译报错

原因是为了避免菱形问题（Diamond Problem）：如果多个父类有同名方法，编译器不知道该用哪一个。 C++ 允许多继承，但 Java 选择了更安全的单继承 + 接口方案。

super 关键字

super 代表父类对象的引用，主要有两种用法：调用父类构造方法和调用父类成员。

① 调用父类构造方法

子类构造方法的第一行必须调用父类构造方法（不写的话编译器默认加 super()）：

public class Animal {
    String name;

    public Animal(String name) {
        this.name = name;
    }
}

public class Dog extends Animal {
    String breed;  // 品种

    public Dog(String name, String breed) {
        super(name);      // 调用父类构造方法，必须放在第一行
        this.breed = breed;
    }
}

注意：如果父类没有无参构造方法，子类必须用 super(参数) 显式调用父类的有参构造， 否则编译报错。

② 调用父类成员

当子类定义了和父类同名的成员时，用 super 区分：

public class Animal {
    public void speak() {
        System.out.println("动物在叫");
    }
}

public class Cat extends Animal {
    @Override
    public void speak() {
        super.speak();              // 先执行父类版本
        System.out.println("喵喵喵"); // 再加上自己的逻辑
    }
}

方法重写（Override）

子类可以重写父类的方法——方法名、参数列表、返回类型完全相同，但方法体不同。 重写让子类表现出自己的行为，这是多态的基础。

重写规则：

  • 方法签名（名称 + 参数）必须完全一致。
  • 返回类型可以是父类方法返回类型的子类（协变返回类型）。
  • 访问权限不能比父类更严格（父类 public，子类不能是 private）。
  • 使用 @Override 注解让编译器帮你检查是否正确重写。
public class Animal {
    public void move() {
        System.out.println("动物在移动");
    }
}

public class Bird extends Animal {
    @Override
    public void move() {
        System.out.println("鸟儿在飞翔");  // 重写为飞翔
    }
}

方法重载 vs 方法重写

这两个概念容易混淆，对比一下：

  • 重载（Overload）：同一个类中，方法名相同、参数不同。编译时决定调用哪个。
  • 重写（Override）：父子类之间，方法签名完全相同。运行时决定调用哪个（多态）。

Object 类 — 所有类的祖先

Java 中所有类都直接或间接继承自 java.lang.Object。 即使你不写 extends，编译器也会自动加上 extends Object。

Object 提供了几个常用方法，子类通常会重写：

public class Person {
    String name;
    int age;

    @Override
    public String toString() {
        return "Person{name='" + name + "', age=" + age + "}";
    }

    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;
        if (obj == null || getClass() != obj.getClass()) return false;
        Person p = (Person) obj;
        return age == p.age && Objects.equals(name, p.name);
    }
}

什么是接口？

接口（Interface）是一种特殊的引用类型，它是行为的契约。 接口只规定"能做什么"，不规定"怎么做"——具体实现交给实现它的类。

打个比方：接口就像一份合同。合同上写了"乙方必须会编程"，但具体怎么写、用什么语言，由签合同的人自己决定。 Java、Python、C++ 程序员都可以签这份合同，各自用自己的方式实现"编程"。

interface 关键字

定义接口使用 interface 关键字：

public interface Flyable {
    // 抽象方法（没有方法体）
    void fly();

    // 默认方法（Java 8+，有方法体）
    default void prepare() {
        System.out.println("准备起飞...");
    }

    // 静态方法（Java 8+）
    static int maxSpeed() {
        return 900;
    }

    // 常量（默认 public static final）
    String TYPE = "飞行生物";
}

接口中的方法默认为 public abstract，变量默认为 public static final（常量）。

implements 关键字

类通过 implements 关键字实现接口，必须重写接口中的所有抽象方法：

public class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("鸟儿扇动翅膀飞行");
    }
}

public class Airplane implements Flyable {
    @Override
    public void fly() {
        System.out.println("飞机靠引擎推力飞行");
    }
}

同一个接口，不同类可以有不同的实现——这就是多态的体现。

一个类可以实现多个接口

Java 虽然不允许多继承类，但允许实现多个接口，用逗号隔开：

public interface Swimmable {
    void swim();
}

public interface Runnable {
    void run();
}

// 实现多个接口
public class Duck implements Flyable, Swimmable, Runnable {
    @Override
    public void fly() {
        System.out.println("鸭子勉强能飞");
    }

    @Override
    public void swim() {
        System.out.println("鸭子擅长游泳");
    }

    @Override
    public void run() {
        System.out.println("鸭子摇摇摆摆地跑");
    }
}

这弥补了 Java 单继承的局限——类只能继承一个父类，但可以实现无数个接口。 所以常说：继承是"是什么"（is-a），接口是"能做什么"（can-do）。

接口可以继承接口

接口之间也用 extends 继承，并且支持多继承：

public interface Printable {
    void print();
}

public interface Scannable {
    void scan();
}

// 接口多继承
public interface MultiFunction extends Printable, Scannable {
    void fax();
}

// 实现类需要重写所有方法
public class OfficeMachine implements MultiFunction {
    @Override public void print() { System.out.println("打印中..."); }
    @Override public void scan()  { System.out.println("扫描中..."); }
    @Override public void fax()   { System.out.println("传真中..."); }
}

接口 vs 抽象类

这是面试高频问题，有必要搞清楚：

对比项 抽象类 接口
关键字 abstract class interface
构造方法 有 没有
成员变量 可以是各种类型 只能 public static final 常量
方法 可以有抽象方法和具体方法 抽象方法 + default/static 方法
多继承 不支持 支持（接口可以继承多个接口）
适用场景 有共同父类的"是什么"关系 跨不同类层次的"能做什么"能力

一句话口诀：共用属性写继承，共用行为写接口。

多态 — 继承与接口的共同目标

无论是继承还是接口，最终都在服务于多态（Polymorphism）—— 同一个类型的引用可以指向不同子类 / 实现类的对象，调用方法时表现出不同行为。 

// 继承的多态
Animal a1 = new Dog();
Animal a2 = new Cat();
a1.speak();  // 狗叫
a2.speak();  // 猫叫

// 接口的多态
Flyable f1 = new Bird();
Flyable f2 = new Airplane();
f1.fly();    // 鸟儿扇动翅膀飞行
f2.fly();    // 飞机靠引擎推力飞行

这种写法的好处是面向抽象编程——代码依赖的是父类 / 接口类型， 而不是具体的子类。将来替换实现时，调用方的代码完全不用改。

小结

今天我们学习了 Java 面向对象编程中最核心的两个机制：

  • 继承（extends）：子类复用父类代码，形成 "is-a" 层次关系。Java 单继承，一个类只有一个爹。
  • super：在子类中引用父类成员或调用父类构造方法。
  • 方法重写（Override）：子类用自己的方式实现父类方法。
  • 接口（interface / implements）：定义行为契约，一个类可以实现多个接口，弥补单继承的局限。
  • 多态：继承和接口共同支撑多态，让代码更灵活、更易扩展。

掌握继承和接口，是写出优雅 Java 代码的关键。 记住两条原则：继承看"是什么"（狗是动物），接口看"能做什么"（鸟能飞）。 下一课我们将深入探讨多态的底层原理。
