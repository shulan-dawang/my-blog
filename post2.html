Java 的类与对象

2026 年 5 月 11 日

什么是类？

类（Class）是 Java 中最核心的概念。你可以把类理解为一张蓝图或模板—— 它定义了某一类事物的属性（数据）和行为（方法），但本身不能直接使用。

打个比方：类是「学生」这个概念，它描述了学生有名字、学号、成绩这些属性，以及上课、考试这些行为。

什么是对象？

对象（Object）是类的一个具体实例。类只有被 new 出来之后， 才变成一个实实在在可以操作的对象。

继续上面的比方：对象就是「张三」这个具体的学生——他有具体的名字"张三"、学号"2026001"、成绩 85 分。

一句话总结：类是对象的模板，对象是类的实例。

定义一个类

我们先定义一个简单的 Student 类：

public class Student {
    // 属性（成员变量）
    String name;
    int age;

    // 方法（行为）
    public void study() {
        System.out.println(name + " 正在学习 Java！");
    }
}
  • name 和 age 是成员变量，描述对象的状态。
  • study() 是成员方法，描述对象能做什么。

创建并使用对象

有了 Student 类，我们用 new 关键字来创建对象：

public class TestStudent {
    public static void main(String[] args) {
        // 创建对象
        Student s1 = new Student();

        // 给属性赋值
        s1.name = "张三";
        s1.age = 18;

        // 调用方法
        s1.study();  // 输出：张三 正在学习 Java！

        // 再创建一个对象
        Student s2 = new Student();
        s2.name = "李四";
        s2.age = 20;
        s2.study();  // 输出：李四 正在学习 Java！
    }
}

s1 和 s2 是同一个类的两个独立对象， 互不影响。每个对象都有自己的 name 和 age。

类与对象的内存关系

理解内存模型有助于真正掌握面向对象：

  • 类是存储在方法区（Method Area）中的模板信息。
  • 当执行 new Student() 时，JVM 在堆内存（Heap）中分配空间存放对象的数据。
  • 变量 s1 本身存在栈内存（Stack）中，它保存的是对象在堆中的地址（引用）。

成员变量有默认值

如果创建对象后不给成员变量赋值，Java 会自动给一个默认值：

Student s = new Student();
System.out.println(s.name);  // null  （引用类型默认值）
System.out.println(s.age);   // 0     （int 默认值）

常见默认值：整数是 0，浮点数是 0.0，布尔是 false，引用类型是 null。

小结

今天我们学习了 Java 面向对象编程的基础——类和对象。 类定义了属性和行为，对象是类实例化后的具体存在。 记住核心公式：类 → new → 对象。下一课我们将学习构造方法，让创建对象的过程更加优雅。
